0. 音響基礎と担当楽器の特性調査（設計前のリサーチ）

なぜ先にこれをやる？
  • 音色設計は「物理的に何がどうなっているか」を理解していないと、パラメータをいじっても勘頼みになる。
  • フーリエ変換 → 倍音構造 → 実測スペクトル の順で「目標の音が物理的にどんな構成か」をつかんでから設計に入ると、根拠を持って数値を決められる。
  • ここで得た知見は 2 章（音源生成方式）・5 章（クラリネット特有の検討） で直接使う。クラリネットは奇数倍音優勢という明確な特徴があるので、実測でそれを確認するのが特に重要。

「任意の波形は正弦波の和として表せる（フーリエ級数）」／「時間波形 ↔ 周波数スペクトルの変換」／「FFT は離散版の高速アルゴリズム」の 3 点を 3〜5 行で説明できるように。
調べ先：Wikipedia「フーリエ変換」「高速フーリエ変換」、YouTube の直感的解説動画（3Blue1Brown など）も有効。

基音 (fundamental) と倍音 (harmonics / overtones)、整数倍の関係、偶数倍音 vs 奇数倍音が音色に与える影響。代表波形との対応（サイン波＝倍音なし／のこぎり波＝全倍音／矩形波＝奇数倍音のみ）も書く。クラリネットは矩形波の倍音構成に似ることを意識。

クラリネットはマウスピースのリード（薄い竹片）が振動して音を出し、片端閉・片端開の円筒管として振動する。この境界条件のため奇数倍音のみが定在波モードとして強調される（偶数倍音は弱い）。これがフルートとの決定的な違い。
リードの「擦れ音」的な微小ノイズも音色特徴。
調べ先：Wikipedia「クラリネット」「閉管（気柱振動）」、『音響学入門』類。

YouTube / freesound.org / 大学の音響資料サイト 等で C4 相当など単音の音源を確保。できればフルートの録音も並べて用意し、後で比較できるようにする。使った音源名・URL を下に記録。

手順：

  1. 音源を Audacity で開く
  2. 安定している区間（アタック/リリースを除く中間の 0.5〜1 秒）を選択
  3. メニュー 解析 → スペクトル（サイズ 4096〜8192、関数 Hann）
  4. ピーク位置にカーソルを合わせて周波数と dB を読む
  5. 各倍音のレベルを下の表に記入。偶数と奇数の差 に注目すること。

💡 コツと注意点（調べ方）
  • MP3 より WAV / FLAC 推奨：MP3 は高次倍音が鈍ることがある。YouTube の音源は高圧縮なので、解析用にロスレス音源が手に入ればそちらを優先。
  • エフェクト（リバーブ／EQ）付きは避ける：スペクトルが歪んで本来の倍音構成が見えにくくなる。ソロのクリーン録音を選ぶ。
  • dB は相対値：表示される dB は曲全体の最大振幅を 0 dB とした基準値。絶対値は気にせず、基音比（各倍音が基音より何 dB 低いか）で見れば OK。
  • ピッチのズレ：演奏のチューニングが正確でないと ×2, ×3 の理論周波数に完全一致しない。ピーク付近の値で妥協してよい。
🔧 Audacity 以外の解析ツール（選択肢）
  • Python (numpy + scipy.fft + matplotlib)：AI にスクリプトを書かせて python fft.py sample.wav で一発解析。表作成まで自動化できる。厳密な数値が欲しい場合に。
  • Sonic Visualiser：Audacity より見やすいスペクトログラム表示ができる無料ツール。
  • SPAN（無料 VST プラグイン）：DAW と併用するならこれ。
