# sync

**全楽器で同期演奏するための方針** をまとめておくディレクトリです。
1 つの楽器に閉じない、 **チーム全員で守る決まりごと** はここに置きます。

## 中身

```
sync/
├── README.md              ← このファイル
├── 同期方針計画書.html     ← 同期方針の本体 (ブラウザで開いて読む)
├── 同期方針計画書.md       ← 同じ内容の Markdown 版 (GitHub 上で読みやすい)
├── arduino/               ← 全楽器で共通の Arduino コード (マスター時計・共通プロトコル等)
├── processing/            ← 全楽器で共通の Processing コード (指揮者 UI 等)
└── memo/                  ← 同期に関する自由スペース (検討メモ・図など)
```

## それぞれの役割

- **`同期方針計画書.html` / `同期方針計画書.md`**
  テンポ・タイミング・通信フォーマット (シリアル / MIDI など) ・スタート合図の取り方など、 **全楽器が共通して従うルール** が書いてあります。
  中身は同じものの 2 形式です。お好みで使い分けてください:
  - **`.html`** … ブラウザで開くと図やレイアウト込みで綺麗に見える
  - **`.md`** … GitHub のページ上でそのまま読める / `git diff` で変更箇所が見える / エディタで編集しやすい

  Arduino と Processing を実装するときは、まずここを読んで仕様に合わせてください。
  内容を更新するときは **`.md` を編集 → `.html` も同じ内容で更新** すると 2 つがズレずに済みます。

- **`arduino/`**
  **全楽器で共通して使う** Arduino コードを置く場所です。 1 楽器に閉じない処理 — 例えばマスター時計、テンポ配信、共通のメッセージプロトコル定義 (ヘッダファイル) など — はここへ。
  各楽器固有の Arduino コードは引き続き楽器ディレクトリ側 (`clarinet/arduino/` など) に置いてください。

- **`processing/`**
  **全楽器で共通して使う** Processing コードを置く場所です。 例: 指揮者役の UI、全楽器に一斉スタートを送るコントローラなど。
  各楽器固有の Processing コードは楽器ディレクトリ側 (`clarinet/processing/` など) に置きます。

- **`memo/`**
  同期まわりの **自由スペース**。配線図、シーケンス図、検討中のプロトコル案、参考にした記事のリンク集など、 「同期に関係はあるけど仕様書に載せるほどではない」 ものを気軽に置く場所です。

## 自由に追加してOK
このディレクトリも楽器ディレクトリと同じく、 **同期に関係するものは何を足してもOK** です。

- 同期方針計画書を更新したら、上書き保存してコミット
- 図やフローチャートが増えたら `sync/diagrams/` のようなサブディレクトリを作って入れる
- 通信プロトコルの仕様書を独立させたくなったら `sync/protocol.md` のように追加

迷ったら、 **「他の楽器担当も読むかどうか」** を基準にしてください。読むなら `sync/` に置く、自分しか読まないなら自分の楽器ディレクトリに置く、で切り分けられます。

## 共通の決まりごと

- このディレクトリの内容を変えるときは、必ず Issue を立ててから直してください。 **同期方針が変わると全楽器の実装に影響する** ためです。
- 作業の進め方 (Issue / PR / ブランチの切り方) はルートの [`../README.md`](../README.md) を読んでください。
