0. 音響基礎と担当楽器の特性調査（設計前のリサーチ）

なぜ先にこれをやる？
  • 音色設計は「物理的に何がどうなっているか」を理解していないと、パラメータをいじっても勘頼みになる。
  • フーリエ変換 → 倍音構造 → 実測スペクトル の順で「目標の音が物理的にどんな構成か」をつかんでから設計に入ると、根拠を持って数値を決められる。
  • ここで得た知見は 2 章（音源生成方式、特に 2-1 倍音構成） で直接使う。オルガンは 加算合成 的な楽器なので、倍音ごとのレベル測定が設計そのものに直結する。

「任意の波形は正弦波の和として表せる（フーリエ級数）」／「時間波形 ↔ 周波数スペクトルの変換」／「FFT は離散版の高速アルゴリズム」の 3 点を 3〜5 行で説明できるように。
調べ先：Wikipedia「フーリエ変換」「高速フーリエ変換」、YouTube の直感的解説動画（3Blue1Brown など）も有効。

基音 (fundamental) と倍音 (harmonics / overtones)、整数倍の関係、偶数／奇数倍音が音色に与える影響。
さらに、フーリエの考え方は「複数の正弦波を足し合わせて任意の音を作れる」という加算合成（additive synthesis）の根拠でもあり、オルガン（特にハモンド）の設計思想そのもの、という点まで書く。

オルガンは種類で原理が違う。

  • パイプオルガン：1 本のパイプが 1 つの倍音（厳密には基音中心のスペクトル）を担当。ストップ（パイプ群）を重ねて音色を作る＝物理的な加算合成。
  • ハモンドオルガン：9 本のドローバーで 9 つの倍音（Sub, 1, 2, 3, 4, 5, 6, 8 など）のレベルを独立制御する加算方式。トーンホイールで正弦波に近い音を発生させる。
  • 電子オルガン：デジタル合成／サンプル再生。
どの方向を目指すかで倍音設計が変わる点まで書く。
調べ先：Wikipedia「パイプオルガン」「ハモンドオルガン」、ドローバーの解説記事。

目指す方向（パイプ／ハモンド／電子）に合わせて YouTube / freesound.org / サンプルライブラリ から単音を確保。理想的にはパイプオルガンとハモンドオルガンを両方入手して FFT を比べると違いが明瞭にわかる。使った音源名・URL を下に記録。

手順：

  1. 音源を Audacity で開く
  2. 安定している区間（鍵を押している中間の 0.5〜1 秒）を選択
  3. メニュー 解析 → スペクトル（サイズ 4096〜8192、関数 Hann）
  4. ピーク位置にカーソルを合わせて周波数と dB を読む
  5. 各倍音のレベルを下の表に記入。どの倍音が強いか の分布がそのまま加算合成の設計になる。

💡 コツと注意点（調べ方）
  • MP3 より WAV / FLAC 推奨：MP3 は高次倍音が鈍ることがある。YouTube の音源は高圧縮なので、解析用にロスレス音源が手に入ればそちらを優先。
  • エフェクト（リバーブ／EQ）付きは避ける：オルガンは教会録音でリバーブが強烈にかかっていることが多い。ドライ（残響なし）の録音を選ぶとスペクトルが素直に出る。
  • dB は相対値：表示される dB は曲全体の最大振幅を 0 dB とした基準値。絶対値は気にせず、基音比（各倍音が基音より何 dB 低いか）で見れば OK。
  • ピッチのズレ：演奏のチューニングが正確でないと ×2, ×3 の理論周波数に完全一致しない。ピーク付近の値で妥協してよい。

2. 音源生成方式

前提：波形は Processing 側で合成する
Arduino 側は音階データの送信のみを担当し、波形合成は Processing で行う想定。オルガンは和音を扱うため、Arduino 側（tone() / PWM など）で複数倍音を加算しつつ和音も鳴らすのは現実的でない、というのがこの分担の根拠。3 人で同じ前提になっているかだけ最初に確認してください。

オルガン（特にハモンド）は「サイン波を複数重ねる＝加算合成」が基本。ベースをサイン波に置くか、既成波形（のこぎり／矩形）をローパスで整形するかなど、採用する方式を決める。

何倍音までが目立つ？ 偶数／奇数で偏りはある？ ハモンドの「ドローバー」方式はどういう思想？

5. オルガン特有の検討事項

オルガンは押している間ずっと同じ音量で鳴るのが特徴。ADSR の設定（Sustain 高め／Decay 抑制）や、長時間発音でも音量が揺れない再生方法を整理する。

加算合成したサイン波を単純に重ねると「電子っぽさ」が出やすい。デチューン（わずかな周波数ズレ）・位相ランダム化・残響などの候補から、採用するものを選ぶ。実験して判断。

フルート・クラリネットより実装が楽な部分（例：息ノイズがいらない／ビブラート不要）と、オルガンだけが難しい部分（例：和音・ボイス管理・合算音量）を並べて整理する。

時間がないときに何だけ残せば「オルガンっぽく」聞こえるか。例：サイン波 3 本（×1, ×2, ×3）＋ Sustain 最大 ＋ クリック回避用の短い Attack/Release、くらいから試す。
