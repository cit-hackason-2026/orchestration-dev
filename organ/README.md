# organ

オルガン担当のディレクトリです。

## 中身

```
organ/
├── README.md                       ← このファイル
├── organ_design.html            ← オルガンの設計資料 (ブラウザで開いて読む)
├── organ_design_answers.json    ← 自分で作った設計シートの回答などを入れる JSON
├── arduino/                        ← Arduino スケッチ (.ino)
├── processing/                     ← Processing スケッチ (.pde)
├── sound/                          ← 音源と FFT スペクトル
│   ├── sample/                     ← 目標とする音 (本物の楽器の録音など)
│   ├── output/                     ← 実際にこの楽器で作れた音
│   └── spectrum/                   ← FFT で解析したスペクトル画像
└── memo/                           ← なんでも置いてOKの自由スペース
```

## それぞれの役割

- **`organ_design.html`**
  オルガンの仕組み・使う部品・回路・音の出し方など、設計のすべてが書いてあります。
  まずはこれをブラウザで開いて読んでください。

- **`organ_design_answers.json`**
  設計シート (`organ_design.html`) を埋めた結果や、自分でまとめた設定値などを書き出した JSON を **ここに置きます**。
  最初は空 (`{}`) になっています。書き出しツールが出してくれた JSON を、丸ごとこのファイルに貼り付けて上書き保存してください。
  別の名前で複数作りたい場合 (例: `organ_pinout.json` など) もこのディレクトリに置いてOKです。

- **`arduino/`**
  実機を制御する Arduino のコードを置きます。 `.ino` ファイルを Arduino IDE で開いて書き込みます。
  スケッチ名はディレクトリ名と揃えるのが Arduino IDE の流儀なので、 `arduino/organ/organ.ino` のように作るのがおすすめです。

- **`processing/`**
  PC 側で楽譜・タイミング・UI を扱う Processing のコードを置きます。 `.pde` ファイルを Processing IDE で開きます。
  Arduino とはシリアル通信 (USB) で繋ぐ想定です。具体的な通信フォーマットは[`../sync/同期方針計画書.html`](../sync/同期方針計画書.html) を参照。

- **`sound/`**
  音に関するものをまとめておくディレクトリです。中は 3 つに分かれています:
  - **`sound/sample/`** … **目標とする音** をここに入れます。本物のオルガンの録音や、お手本にしたい音源 (`.wav` / `.mp3` など)。「これに近付けたい」というゴールを示すファイルです。
  - **`sound/output/`** … **実際にこの楽器で作れた音** をここに入れます。試作の段階ごとに `output_v1.wav`, `output_v2.wav` のように残しておくと、改善の経過が分かって後で発表のときも便利です。
  - **`sound/spectrum/`** … 上の音を **FFT で解析した結果のスペクトル画像** (`.png` など) を入れます。 `sample_fft.png`, `output_v1_fft.png` のように、元になった音と対応が分かる名前にしてください。

  > FFT の解析は Processing の FFT ライブラリ、 Audacity の「スペクトル表示」、 Python (`numpy.fft` + `matplotlib`) など、好きなツールでOK。出力した画像をここに置けば十分です。

- **`memo/`**
  気軽に何でも置いていい自由スペース。手書きメモのスキャン、参考にした図、買った部品のリスト、配線写真、検討中のアイデアなど、 **「このディレクトリにあると後で見直せて助かる」** ものを入れておく場所です。
  ここの中身はチームで使うコードではないので、ファイル名やフォルダ分けも自由でOK。

## 自由に追加してOK

このディレクトリは **オルガン担当のあなたのためのスペース** です。
上の構成は最低限の枠で、 **自分が必要だと思うファイル・フォルダは自由に追加して構いません。**

- 上の枠に当てはまるものは、できればその場所に置く (例: コードは `arduino/` か `processing/` へ)
- どこにも当てはまらないラフなものは `memo/` に置く
- 大きなジャンルが新しくできたら、独自のサブディレクトリを作ってもOK
  例: `docs/` (ちゃんとした文書), `tests/` (動作確認用データ), `images/` (写真) など

迷ったら **「自分が一週間後に見て分かるか」** を基準にしてください。

## 共通の決まりごと

- 全楽器で同期して演奏するため、**タイミング・通信・テンポなどの方針** はルートの [`../sync/同期方針計画書.html`](../sync/同期方針計画書.html) に従ってください。
- 作業の進め方 (Issue / PR / ブランチの切り方) はルートの [`../README.md`](../README.md) を読んでください。

## 困ったとき

- 設計の意図が分からない → `organ_design.html` を読み直す → それでも不明なら Issue に `question` ラベルで質問を立てる
- 他の楽器との同期が合わない → [`../sync/同期方針計画書.html`](../sync/同期方針計画書.html) を見て、自分の実装が方針通りか確認
